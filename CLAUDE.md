## ✍️ メディア編集方針

### 取り扱わないテーマ（編集方針として対象外）
- **医療・健康に関する記事は対象外とする**
- 理由：記事を読んだ飼い主さんが誤った行動をとるリスクがあるため
- 病気、症状、治療法、薬、食事療法、アレルギー対応など医療判断に関わるテーマは扱わない
- 健康情報が必要な文脈では「かかりつけの獣医師に相談してください」と案内するにとどめる

### コンテンツの3レイヤー（2026-03-14〜）

AI時代のメディア生存戦略として、**一次体験に基づくエンターテインメント**と**書き手の人格・語り口による共感**を最優先する。

| レイヤー | 目標比率 | 内容 |
|---|---|---|
| エンタメ | 50%以上 | 柴犬あるある、飼い主あるある、笑える実体験。AIに絶対書けない領域 |
| 体験ストーリー×情報 | 30% | 実用情報を体験談で包む。「○○してみたらこうなった」の語り口 |
| 情報 | 20%以下 | SEO流入用の入口記事。書く場合もエンタメ要素を加える |

### トーン＆ボイス
- **ポジティブ・クスッと笑える・誰も傷つかない** を3原則とする
- 飼い主の一人称視点が基本
- 柴犬のリアクションを具体的に描写する（「固まった」「首をかしげた」「ため息をついた」）
- 読者に語りかける文体（「〜ですよね」「〜ありませんか？」）
- 情報を羅列せず、ストーリーの中に織り込む
- 読者層：20〜40代の柴犬オーナー・柴犬に興味がある人

### タイトルの付け方
- 感情や好奇心を刺激しつつ、**検索キーワードを含める**
- タイトル：**40文字以内**
- 「地域名」「商品名」「犬種名」「具体的な状況」を含め、検索者が使う言葉をタイトルに入れる
- ❌「生田緑地で柴犬と散歩してきた」→ ✅「生田緑地は犬連れ散歩に最高だった【土の道が多くて柴犬向き】」

### 記事構成の基本
- 文字数：2,000〜3,000字
- **冒頭に実用情報を地の文として追加する**（まとめボックス・囲み枠・箇条書きブロックは禁止）
  - 冒頭の書き出し1〜2文はそのまま残す → 「先にポイントだけ。」等の接続（毎回変える） → 実用情報を2〜4文の地の文で流す → 元の本文に戻る
  - お出かけ記事：犬連れOKか・混雑状況・一言感想を冒頭で。施設情報（住所等）は記事末尾に
  - レビュー記事：商品名・価格帯・一言結論を冒頭で。スペック詳細は本文中の商品紹介パートに
  - あるある記事：省略可（いきなり本文でOK）
  - ❌ まとめボックス・「この記事でわかること」・SWELLキャプション付きボックス → ✅ 地の文に自然に溶け込ませる
- 構成：冒頭（書き出し＋実用情報） → 本文（H2×3〜4） → まとめ
- 画像alt属性は必ず設定する

### 派生記事の考え方
- 1つの体験に複数テーマが含まれる場合、それぞれを独立記事として切り出す
- 元の日記はそのまま残し、派生記事を別途作成して相互リンクする
- 例：キャンプ日記 → 派生①テントのレビュー記事、派生②キャンプ場のレビュー記事
- **派生記事のトーン**: まーくん一人称の体験ベースで、日記よりは情報寄り。企業サイトのような三人称にしない。「友達に聞かれたから詳しく教えてあげる」くらいの距離感。夫婦の掛け合いは無理に入れなくてOK

### マネタイズ意識（リンクはまだ入れない）
- 記事の構造として収益導線を意識して書く
- レビュー記事で商品名・価格・使用感を書いておけば、後からリンクを差し込むだけで収益化できる状態にしておく
- 実際のアフィリエイトリンク設置は月間1万PV到達後

### SEOルール
- メタディスクリプション：120文字以内
- H2見出し：3〜5個、各見出しにキーワードを含める
- 内部リンク：記事内に最低2本設置

## 🤖 自動化パイプライン仕様

### Phase 1：トレンド収集（毎日 9:00）
```
入力: カテゴリキーワード（柴犬, 犬ケア, ペット用品 等）
出力: ネタ候補リスト（タイトル案×10件）
保存先: Notion「📝 ネタ帳」データベース（Notion MCP経由）
ステータス管理: 💡アイデア → ✅採用 → 📝執筆中 → 🚀公開済 / ❌不採用
```

### Phase 2：記事生成（毎日 10:00）
```
入力: ネタ帳から優先度上位1件
出力: 記事本文（Markdown）+ X投稿文×3 + Instagramキャプション×1
保存先: 01_編集部/02_執筆・原稿/YYYY-MM-DD_[記事タイトル].md
```

### Phase 3：品質チェック（生成直後）
```
チェック項目:
  - SEOスコア（タイトル・見出し・キーワード密度）
  - 文字数（2000字以上）
  - トーン確認（温かみ・柴犬目線）
  - 重複チェック（過去記事との類似度）
出力: 品質レポート（合格 / 要修正）
```

### Phase 4：公開（WordPress REST API）
```
エンドポイント: https://shiba-with.com/wp-json/wp/v2/posts
認証: Application Password（環境変数から取得）
投稿ステータス: draft（レビュー後に手動公開 or 自動公開）
```

### Phase 5：分析・改善（毎週月曜 9:00）
```
入力: GA4データ（過去7日間）
出力: 週次レポート + 次週コンテンツ戦略提案
保存先: _archive/03_集客部/03_アクセス解析・レポート/YYYY-WXX_週次レポート.md
```

## ✅ 承認不要（事後報告のみ・即時自動実行）

コストが発生しない以下の操作は、ユーザーの許可を待たずに即実行する：

### ファイル操作
- ローカルファイルの作成・読み込み・編集・削除・整理
- カスタムコマンド・スキルファイルの作成・更新
- CLAUDE.md・メモリファイルの更新

### コード・開発
- コード解析・静的テスト・ビルド・Lint等のコマンド実行
- Bashコマンド全般
- npm/pip等のパッケージインストール（ローカル環境）
- Git操作（commit・branch作成・ローカルでのmerge・push）
- **作業完了時は必ず git commit + push を実行し、GitHubを常に最新に保つ**

### WordPress・コンテンツ
- WordPress REST APIへの通信（記事の取得・下書き作成・更新・公開記事の更新）
- **ローカル記事の修正後 → WordPressへの自動反映**
- 記事の表現修正・構成リライト
- /editorial-check, /pet-health-check の自動実行
- 内部リンクの修正・差し替え

### エラー処理・フォールバック
- Fetch時のエラーハンドリング（リトライ・スキップ・次の候補へ移行）
- ソースが見つからない場合 → 「かかりつけの獣医師に相談してください」への置き換え
- 解決可能なエラーは自己修復して続行

### 並行処理
- 複数タスクの並行実行（Agent・バックグラウンドタスク）
- 依存関係のないタスクの同時起動

## ⛔ 承認必須（コスト発生リスクがある操作のみ）

以下の操作は実行前に必ずユーザーに確認を取ること：

| 操作 | 理由 |
|---|---|
| **外部の有料APIへの新規接続** | 課金が発生する可能性 |
| **課金が発生する操作全般** | AWS・GCP・有料SaaS等 |
| **WordPressの設定変更** | テーマ・プラグイン・パーマリンク等、サイト全体への影響 |
| **サイト全体に影響する一括変更** | 全記事のカテゴリ変更・一括ステータス変更等 |
| ~~Git push~~ | ~~承認必須から除外済み（承認不要で自動実行）~~ |
| **不可逆な本番環境の操作** | データベース変更・ドメイン設定等 |
